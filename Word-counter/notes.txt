Aakash, text-align sirf inline elements (jaise text, <span>, <a>, <img>) ko align karta hai. Yeh kab kaam nahi karta, uske liye ye rules apne notes mein likh lo:
1. Block Elements ko Center karne ke liye 
text-align: center kisi block element (jaise <div>, <section>) ko khud center nahi karta, sirf uske andar ke text ko center karta hai.
Rule: Agar poore box/div ko center karna hai, to margin: 0 auto; ka use karein aur element ki width define karein. 

2. Inline Elements ki Width
Agar element (jaise <span>) sirf apne content jitna hi bada hai, to text-align ka asar nahi dikhega kyunki hilne ke liye jagah hi nahi hai.
Rule: Parent element par text-align lagayein ya element ko display: block ya inline-block banakar width dein. 

3. Absolute Positioning 
Jab aap position: absolute; use karte ho, to element document flow se bahar ho jata hai aur uski width uske content jitni hi reh jati hai.
Rule: Isse center karne ke liye width: 100%; dein ya phir left: 50%; transform: translateX(-50%); ka use karein. 

4. Flexbox aur Grid
Flex container ke andar text-align aksar kaam nahi karta kyunki flex properties (justify-content) alignment ko control karti hain.
Rule: Flexbox mein horizontal centering ke liye justify-content: center; ka use karein. 

5. Floated Elements
Agar element par float: left; ya right; laga hai, to text-align use move nahi kar payega

Here is how it works in the context of this Word Counter project:
Whitespace Removal: When you type into the text area, trim() is used to clean up the string by removing extra spaces, tabs, or newlines at the very beginning and the very end of the input .
Accurate Counting: This ensures that if a user accidentally presses the spacebar before or after their sentence, those empty spaces are not incorrectly counted as words or characters

split(/\s+/) is a JavaScript method used to divide a string into an array of substrings based on a specific pattern called a Regular Expression .
Here is how it works in the context of this Word Counter project:
Regex Pattern (/\s+/): The \s represents any whitespace character (spaces, tabs, newlines). The + signifies that it should match one or more whitespace characters in a row .
Splitting the Text: Instead of just splitting by a single space, this allows the code to treat multiple spaces between words as a single delimiter. This ensures that the word count remains accurate even if a user types extra spaces

Aakash, iska simple karan hai "Exit on Success" rule.
1. if ... else if ... else if (The Chain)
Jab aap saari conditions ko else if se jod dete ho, to JavaScript unhe ek hi group maanti hai.
Rule: Jaise hi koi ek condition True hoti hai, JS uske andar ka code chalata hai aur baaki poori chain ko skip (phenk) deta hai.
Problem in your code: Agar pehli condition InputStr[i] != ' ' true ho gayi (jo ki har letter 'a', 'b', '1' ke liye true hai), to JS niche wale AlphCoute ya numCoute ko check hi nahi karega. Wo wahi se agle loop par chala jayega.
2. if ... if ... else if (The Split)
Yahan aapne do alag-alag groups bana diye hain.
Pehla if: Ye akela hai. Ye hamesha check hoga.
Dusra if-else if: Ye dono ek group hain. Inme se koi ek chalega.
Result: Isliye isme CharCount bhi badhta hai aur phir niche wala group check hokar AlphCoute ya numCoute bhi badhta hai.
Short Notes for Comparison:
Structure	Behavior
Only if ... if	- Dono check honge. - Ek hi character do jagah count ho sakta hai.
if ... else if	- Sirf pehla true wala chalega. - Baaki ignore ho jayenge.
Summary: Agar aap chahte ho ki ek hi character ('a') do alag-alag counters mein gina jaye, to aapko alag-alag if use karne honge.
